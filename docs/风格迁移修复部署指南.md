# 风格迁移功能修复部署指南

## 📋 问题概述

当前风格迁移功能存在**图片变黑**的严重风险，主要原因是值域归一化问题。本指南提供完整的修复和部署方案。

## 🔧 修复内容

### 1. 核心问题

**原代码问题**（`app.py` 第 160 行）：

```python
# ❌ 错误：假设输入总是 [0, 255] 范围
u16 = np.clip(np.round(img / 255.0 * 65535.0), 0, 65535).astype(np.uint16)
```

**问题分析**：

- Diffusers 输出的图像值域可能是 `[0, 1]` 或 `[-1, 1]`
- 如果输入是 `[0, 1]`，除以 255 后会得到 `[0, 0.00392]`
- 最终输出几乎是纯黑色

### 2. 修复方案

**新代码**（`app_fixed.py`）：

```python
def _normalize_array_to_255(arr: np.ndarray, source: str = "unknown") -> np.ndarray:
    """统一将数组归一化到 [0, 255] float32 范围"""

    # 检测值域
    arr_min, arr_max = arr.min(), arr.max()

    if arr_max <= 1.0 and arr_min >= 0.0:
        # [0, 1] 范围 → [0, 255]
        arr = arr * 255.0
    elif arr_min < 0.0 and arr_max <= 1.0:
        # [-1, 1] 范围 → [0, 255]
        arr = (arr + 1.0) * 127.5
    elif arr_max > 255.0:
        # 超出范围，归一化
        arr = (arr - arr_min) / (arr_max - arr_min + 1e-8) * 255.0

    return np.clip(arr, 0.0, 255.0)
```

### 常见问题排查

#### 问题 1: IP-Adapter 未加载

**症状**：

```
[WARNING] Failed to load IP-Adapter: ...
```

**解决**：

```bash
# 检查环境变量
echo $QRAW_IP_ADAPTER_MODEL
echo $QRAW_IP_ADAPTER_WEIGHT

# 手动下载模型
huggingface-cli download h94/IP-Adapter ip-adapter-plus_sdxl_vit-h.safetensors
```

#### 问题 2: 图像仍然变黑

**排查步骤**：

1. 启用调试模式：`export QRAW_DEBUG="1"`
2. 查看日志中的值域信息
3. 检查 `[DEBUG] Input range` 和 `[DEBUG] Output range`
4. 如果输入范围异常，可能是 Diffusers 版本问题

**临时解决**：

```python
# 在 _run_single() 函数末尾添加强制归一化
arr = np.array(image).astype(np.float32)
if arr.max() <= 1.0:
    arr = arr * 255.0
```

#### 问题 3: Tile 融合出现色块

**症状**：大图处理后出现明显的拼接缝或色块

**解决**：

- 增加 `tile_overlap` 参数（建议 128）
- 检查每个 tile 的值域是否一致
- 查看日志中的 `[DEBUG] Normalizing array from tile_X`
