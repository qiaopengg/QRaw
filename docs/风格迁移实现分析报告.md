## 🚨 关键风险：图片变黑问题

### 风险分析

#### 1. 色彩空间转换风险

**问题代码**：

```python
# app.py 第 385 行
image = _ensure_rgb(image)
return np.array(image).astype(np.float32)
```

**风险点**：

- Diffusers 输出的图像范围可能是 `[0, 1]` 或 `[-1, 1]`
- 直接转换为 `float32` 数组时，如果值域是 `[-1, 1]`，负值会在后续处理中被 clip 为 0
- 导致暗部细节完全丢失，图像变黑

#### 2. 归一化问题

**问题代码**：

```python
# app.py 第 160 行
u16 = np.clip(np.round(img / 255.0 * 65535.0), 0, 65535).astype(np.uint16)
```

**风险点**：

- 假设输入 `img` 的值域是 `[0, 255]`
- 但如果 Diffusers 输出的图像值域是 `[0, 1]`，则：
  - `img / 255.0` 会得到 `[0, 0.00392]` 的极小值
  - 乘以 65535 后得到 `[0, 257]`
  - **几乎整张图片都是纯黑色**

#### 3. PIL Image 模式问题

**问题代码**：

```python
# app.py 第 384 行
image = _ensure_rgb(image)
return np.array(image).astype(np.float32)
```

**风险点**：

- Diffusers 的 `images[0]` 返回的是 PIL Image
- PIL Image 的 RGB 模式值域是 `[0, 255]`（uint8）
- 但如果 pipeline 返回的是 numpy 数组或 tensor，值域可能是 `[0, 1]`
- 类型不匹配会导致色彩错误

#### 4. Tile 融合时的精度损失

**问题代码**：

```python
# app.py 第 152 行
region += tile.astype(np.float32) * w
```

**风险点**：

- 如果 tile 的值域不一致（部分 `[0, 1]`，部分 `[0, 255]`）
- 加权融合会导致色彩不连续
- 可能出现局部变黑或色块

### 复现场景

**最可能导致图片变黑的场景**：

1. **高对比度图像**：
   - 暗部细节丰富的图像（如夜景、剪影）
   - Diffusers 输出的暗部值可能接近 0
   - 归一化错误会导致暗部完全变黑

2. **大图分块处理**：
   - 不同 tile 的值域不一致
   - 融合时出现色彩断层
   - 部分区域变黑

3. **Refiner 模式**：
   - Base model 和 Refiner 的输出值域可能不同
   - Latent 空间到 RGB 空间的转换可能出错
   - 导致整体偏暗或变黑

def _run_single(...) -> np.ndarray:
    # ... 现有代码 ...

    image = _ensure_rgb(image)
    arr = np.array(image)

    # 🔧 修复：统一值域到 [0, 255]
    if arr.dtype == np.float32 or arr.dtype == np.float64:
        if arr.max() <= 1.0:
            arr = arr * 255.0
        elif arr.min() < 0:
            arr = (arr + 1.0) * 127.5
    elif arr.dtype == np.uint8:
        arr = arr.astype(np.float32)

    return np.clip(arr, 0.0, 255.0).astype(np.float32)

### 2. 实现色彩对齐（中优先级）

```python
def _apply_luminance_mapping(
    ai_result: np.ndarray,  # [0, 255] float32
    original_img: np.ndarray,  # [0, 255] float32
    strength: float = 0.3
) -> np.ndarray:
    """
    文档第 7.2 节：Luminance-aware Mapping
    """
    # 转换到 LAB 色彩空间
    ai_lab = cv2.cvtColor(ai_result.astype(np.uint8), cv2.COLOR_RGB2LAB).astype(np.float32)
    orig_lab = cv2.cvtColor(original_img.astype(np.uint8), cv2.COLOR_RGB2LAB).astype(np.float32)

    # 仅调整亮度通道（L）
    ai_lab[:, :, 0] = (
        ai_lab[:, :, 0] * (1 - strength) +
        orig_lab[:, :, 0] * strength
    )

    # 转换回 RGB
    result = cv2.cvtColor(ai_lab.astype(np.uint8), cv2.COLOR_LAB2RGB).astype(np.float32)
    return result
```
