## 技术实现

### Python 函数

```python
def _print_progress(percentage: int, description: str) -> None:
    """
    打印带进度条的进度信息
    格式：迁移进度：=========== (1%)
    """
    # 计算进度条长度（总共50个字符）
    bar_length = 50
    filled_length = int(bar_length * percentage / 100)
    bar = '=' * filled_length + ' ' * (bar_length - filled_length)

    # 输出格式化的进度条
    print(f"[PROGRESS] 迁移进度：{bar} ({percentage}%) - {description}")

    # 同时输出简单格式供 Rust 解析
    print(f"[PROGRESS] {percentage}% - {description}")
```

### 使用方法

```python
# 在代码中调用
_print_progress(0, "开始风格迁移...")
_print_progress(50, "处理中...")
_print_progress(100, "完成！")
```

## 进度条特点

### ✅ 优点

1. **可视化直观**
   - 一眼就能看出当前进度
   - 进度条长度直观反映完成度

2. **信息完整**
   - 包含百分比数字
   - 包含当前操作描述
   - 包含详细位置信息（分块处理时）

3. **格式统一**
   - 所有进度信息使用相同格式
   - 易于解析和显示

4. **兼容性好**
   - 同时输出两种格式
   - 前端可以选择显示哪种格式

### 📊 进度条规格

- **总长度：** 50 个字符
- **填充字符：** `=`
- **空白字符：** 空格
- **百分比范围：** 0% - 100%
- **更新频率：** 每个关键阶段

## 常见问题

### Q: 进度条不显示？

**A:** 确保使用的是最新版本的 `app.py` (v0.2.0-complete)

### Q: 进度条格式乱了？

**A:** 检查终端宽度是否足够（建议至少 80 字符宽）

### Q: 只看到百分比没有进度条？

**A:** 前端可能只显示了简单格式，检查 Rust 后端的解析逻辑

### Q: 进度条更新太快看不清？

**A:** 这是正常的，实际使用时每个阶段会持续较长时间
